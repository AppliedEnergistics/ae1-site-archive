Change Log

Version rv4.a - Released Friday, February 1st 2013 10.21PM -06:00

  1. Added Terminal Display to Controller
  2. Added Red stone Controllability to Import and Export Bus. ( ignore / off / on / pulse )
  3. Import and Export Bus now can be configured for single or stack input / output.
  4. Bus speed on single has been reduce to reduce lag.
  5. You can now place empty storage cells into ME Storage cells via interface/chest.
  6. Right clicking a stack in a storage cell now gives you half a stack or half the amount, which ever is smaller.
  7. Fixed issue with terminal contents not properly synchronizing.
  8. "AE no longer replaced MC's Default ItemRenderer" Strikes Back
  9. Import and Export Bus can now both have 8 item slots, the import bus uses these as a filter, while the export bus can export 8 types instead of 1.
  10. Completely redid network discovery code, this should fix the issues with chunk unloading / loading and placing items causing network connectivity issues.
  11. Input / Output Bus now render connections to colored and directional blocks correctly like ME Cable.
  12. Fixed issues with Items which have no NBT data and have no NBT Tag with storage.
  13. After selecting an Item to craft the terminal Gui now Re-Opens.
  14. Massively reduced lag caused by auto-crafting and multiple fronts ( ore dictionary usage / pushing items / full inventories. )
  15. Terminals may now face and and down as well as north/east/south/west.
  16. Added Crafting Terminal.
  17. Thaumcraft3 aspects are now visible in terminal again.
  18. Molecular Assembler Recipe has been changed. ( it is now considerably more expensive. )
  19. Fixed texture for Molecular Assembler, shows up the same in inventory and in world, and now properly has a bottom / top textures.

Version rv3.b - Released Friday, February 1st 2013 10.21PM -06:00

  1. Fixed crash on server side when crafting / receiving buildcraft items ( possibly other mods other items as well ) - ( aka removed logging of display names )

Version rv3.a - Released Friday, February 1st 2013 10.21PM -06:00

  1. Input / Output Bus no longer communicate with ME blocks they face, making it possible to move items from one network to another, via bus -> interface.
  2. Terminals now assume the color of cables they are placed on so different network terminals can be packed closely in a wall.
  3. Added Ore Density / Height Configurations to Config File.
  4. Fixed getPickBlock(...) so you can't pick a certain block...
  5. Terminals Front-Side now glows in the dark, and emits a small amount of light when connected to controller.
  6. When a conflicting terminal is removed, the other controller can now reboot.
  7. A Number of blocks have much better opacity settings.
  8. Fixed bug with network initialization related to order and touching networks.
  9. Improved lighting on Grinder Crank.
  10. AE no longer replaced MC's Default ItemRenderer ( this was a bad move on my part )
  11. Shift Clicking Encoded Patterns now puts them in the correct slot in the interface.
  12. Right Clicking while holding an empty storage cell will now remove the storage cell(block/cluster) from within, and give you back a storage housing, and the original cell(block/cluster). You can insert any cell(block/cluster) into an empty storage house. Only works if you have room for items.
  13. Fixed a bug where simulations caused additional block updates / re-calculations. ( less lag )
  14. Changes to storage network now properly update Terminals.
  15. You can now request crafting of items via Terminal.
  16. Items can now be crafted ( crafting table style ) by using an Assembler and Crafting Patterns.
  17. Items can now be crafted externally ( furnaces / auto crafting tables / anything else for that matter. ) by placing Crafting Patterns inside Interfaces, that are adjacent to inventories or connected pipes these do not need to be valid Patterns you can fudge the recipes.
  18. You can now un-encode an Encoded Pattern by right clicking it in the world ) it returns to being an un-encoded pattern.
  19. Destroying a block of any kind in the network now causes the network to 'reboot' and clear all internal processing. Such as crafting.
  20. Interfaces can now request craft-able items from the network to trigger crafting.
  21. Auto-crafting uses the ore dictionary to retrieve items, so it can pull any ingots regardless of the encoded type.
  22. Auto-crafting properly handles items left on the crafting table, for instance RP2 Handsaws.

Version rv2.d - Released Friday, February 1st 2013 10.21PM -06:00

  1. Fixed Crash Updating Controller from Server.
  2. Pre-Formatted Cells now re-populate the Preformatter when placed inside.

Version rv2.c - Released Friday, February 1st 2013 10.20PM -06:00

  1. Fixed missing Quartz Ore
  2. Fixed issues where AppEng Blocks did not respond to other mods tools and blocks correctly.
  3. Fixed Creative Mode Block Destruction.
  4. Added Option for auto-searching in terminal
  5. Quartz Ore now functions with Fortune Enchants
  6. Added Update Checker. ( option to disable in the config if you dislike this. )
  7. Completely Re-did Block Drops Code.
  8. Quartz Ore is now visible in Creative / NEI.
  9. Quartz Ore is now place-able. ( from NEI / Creative )

The controller is the heart of any ME Network, it also accepts power for the network.
ME Drive
Drive which can store up to 10 Storage Cells, and provides their inventory to the network
ME Chest
A simple single storage cell machine which gives direct access to the contents of a single cell, and provides the cells inventory to the network.
ME Access Terminal
Gives full access to the entire content of an ME Network
ME Crafting Terminal
Terminal which also functions as a Crafting table.
ME Crafting Monitor
Displays information about active crafting jobs, and lets you cancel jobs.
ME Cable
Uncolored cable that can connect ME Devices.
ME Import Bus
Imports items into the network, by removing them from the connected inventories
ME Export Bus
Exprots items out of the network, by placing them into connected inventories
ME Level Emitter
Outputs redstone signals depending on the number of a configured item in the network.
ME Storage Bus
Includes an external inventory such as a chest, into the networks storage.
ME Interface
Acts like a small inventory that can import and export to the network, also used in the crafting process.
ME IO Port
Can load and unload storage cells into and out of the network connected.
ME Wireless Access Point
Provides a range of wireless access at the point where the item is placed.
ME Dark Cable
Toggleable cable which can be used to segment your network into diffrent segments to alter functionality or reduce power drain.
ME Storage Monitor
Can be configured to display the number of a specific item that are stored in the network; can also be upgraded to insert / remove that item.
ME Power Relay
Block which accepts power and relays it to attached networks
ME Transition Plane
Surface which converts anything that touches it into energy form
P2P Tunnel
Point 2 Point Tunnel which can be configured to route items/power/liquids from one point to another in a network.
Spatial Pylon
A Marker used to mark a region of space for capture or deploy ment
ME Spatial IO Port
An IO Port used to capture / deploy regions of space defined by pylons.
ME Pattern Encoder
Used to convert blank patterns, into encoded patterns.
ME Partition Editor
Configure Storage cells to hold diffrent types of items and change how your system stores items.
ME Condenser
Condenses items and blocks into raw energy which can be dispersed or converted into a few select items
ME Bus Memory Card
Memory card that can be used to store, and transfer settings
